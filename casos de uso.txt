Casos de uso

Participantes

Número: 001
Caso de uso: Creación de cuenta tipo participante
Actores: Participante
Descripción: El Participante solicita registrarse, el Sistema muestra un formulario, el Participante ingresa sus datos y confirma el registro, el Sistema muestra el éxito de la operación.
Tipo: Primario

Número: 002
Caso de uso: Ver congresos disponibles
Actores: Participante
Descripción: El Participante solicita consultar la lista de congresos, el Sistema muestra los congresos disponibles.
Tipo: Secundario

Número: 003
Caso de uso: Ver actividades de un congreso
Actores: Participante
Descripción: El Participante selecciona un congreso, el Sistema muestra el listado de actividades.
Tipo: Secundario

Número: 004
Caso de uso: Inscribirse a congreso
Actores: Participante
Descripción: El Participante solicita inscribirse a un congreso, selecciona sus tipos de participación y el Sistema registra la inscripción pendiente de pago.
Tipo: Primario

Número: 005
Caso de uso: Pagar inscripción
Actores: Participante
Descripción: El Participante confirma el pago de una inscripción utilizando saldo de la cartera digital, el Sistema descuenta el monto, aplica la comisión y confirma el éxito de la transacción.
Tipo: Primario

Número: 006
Caso de uso: Enviar trabajo a congreso
Actores: Participante
Descripción: El Participante envía un trabajo (ponencia o taller) para un congreso con convocatoria abierta, el Sistema almacena la propuesta para su evaluación.
Tipo: Primario

Número: 007
Caso de uso: Recargar cartera digital
Actores: Participante
Descripción: El Participante solicita recargar su cartera digital indicando el monto y la fecha, el Sistema actualiza el saldo disponible.
Tipo: Primario

Número: 008
Caso de uso: Reservar cupo a actividad tipo taller
Actores: Participante
Descripción: El Participante solicita reservar un cupo en un taller, el Sistema valida disponibilidad y confirma la reserva.
Tipo: Primario

Número: 009
Caso de uso: Generar certificado
Actores: Participante
Descripción: El Participante solicita su diploma de participación, el Sistema valida los criterios y genera el certificado.
Tipo: Secundario

Administradores de congreso

Número: 010
Caso de uso: Crear congreso
Actores: Administrador de congreso
Descripción: El Administrador de congreso crea un nuevo congreso asociado a su institución, definiendo precio, fechas, ubicación e información básica.
Tipo: Primario

Número: 011
Caso de uso: Abrir convocatoria
Actores: Administrador de congreso
Descripción: El Administrador de congreso abre una convocatoria para recepción de trabajos en el congreso, el Sistema habilita el registro de propuestas.
Tipo: Primario

Número: 012
Caso de uso: Cerrar convocatoria
Actores: Administrador de congreso
Descripción: El Administrador de congreso cierra la convocatoria de recepción de trabajos, el Sistema bloquea el envío de nuevas propuestas.
Tipo: Secundario

Número: 013
Caso de uso: Evaluar trabajo
Actores: Participante
Descripción: El comité científico revisa un trabajo enviado, el Sistema registra la decisión de aprobar o rechazar.
Tipo: Primario

Número: 014

Tipo: Primario

Número: 015
Caso de uso: Crear cuenta invitado

Tipo: Secundario

Número: 016

Tipo: Secundario

Número: 020
